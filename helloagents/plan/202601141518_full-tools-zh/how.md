# 方案
- 路径与策略：
  - 以 `locales/en.yml` / `locales/zh.yml` 为基准，检查 `src/tools/**` 中硬编码英文（字段/占位/提示/错误/示例）。
  - 对列表型或复杂文本，若示例需保持英文格式（如 regex、URL、代码片段），保留原字符并增加中文说明/标签。
  - 批处理优先级：通用组件 & 高频工具 > 其他工具；按分类或批次提交，避免一次性大改难以回溯。
- 产出：
  - 中英文本键值（保持英文基线）写入 locales；组件引用改为 `$t`。
  - 必要时同步测试断言（如标题/提示文本）到新文案。
  - 最终巡检，确保无明显残留硬编码英文。
- 验证：
  - 至少运行 `pnpm lint`；如涉及测试文案调整，抽样跑相关单测/e2e。
  - 手动抽查若干工具页（每类至少 1-2 个）和公共区域（搜索、命令面板）。

# 任务拆解（批次化）
1) 基线与脚本：搜索硬编码英文（rg + 关键词）、建立文案键名规范；先处理公共组件/常用工具。
2) 批次1：Crypto + Converter 类工具文案/示例本地化并写入 locales。
3) 批次2：Web + Development + Network 文案/示例本地化。
4) 批次3：Math/Measurement/Text/Data 文案/示例本地化。
5) 验证：lint，抽样测试或手动检查；补漏。
