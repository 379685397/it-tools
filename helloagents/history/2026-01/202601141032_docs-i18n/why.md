# 背景
- 项目提供 90+ IT 小工具，但仓库缺少集中功能文档与索引，用户难以快速了解覆盖范围与入口。
- 站内存在部分写死的英文文案和 PWA manifest 语言固定为 fr-FR，中文/多语言体验不一致。

# 目标
- 产出一份覆盖现有工具分类与功能简介的文档，并在 README/站内入口可发现。
- 梳理写死英文的前端展示文案，统一中文（或纳入 i18n）并修正 PWA 语言设置。
- 给出新增工具/文档的维护流程与质量基线，降低后续维护成本。

# 范围
- 文档：新增工具目录文档（Markdown），在 README/站内入口挂载。
- 站点：为工具指南提供可访问入口（导航或首页链接）。
- 文案：排查常用入口/布局/页面的硬编码英文，移入 locales，补充 zh 文案；调整 PWA manifest 语言。

# 非目标
- 不做后端/接口改动；不做工具功能级别的重构。

# 风险与影响
- i18n 覆盖不全可能导致缺失键或回退英文；需核验关键路径。
- 新文档需与工具列表同步，后续新增工具需更新指南（需在流程中强调）。
