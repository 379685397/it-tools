# 模块：i18n（国际化）

## 1. 目标

- 提供整站多语言能力，默认中文展示，并保留语言切换与英文回退。
- 将工具页与公共组件中的提示、标签、占位、报错与示例从硬编码迁移到语言包中，提升一致性与可维护性。
- 当前产品形态仅对用户开放中文（`zh`）与英文（`en`）两种语言选择。

---

## 2. 关键文件

- i18n 插件：`src/plugins/i18n.plugin.ts`
- 语言选择器：`src/modules/i18n/components/locale-selector.vue`
- 语言包：`locales/*.yml`

---

## 3. 约定与最佳实践

### 3.1 键名组织

- 工具元信息与工具页文案：`tools.<tool-key>.*`
- 分类名称：`tools.categories.*`
- 公共组件默认文案：`ui.*`

### 3.2 工具元信息的特殊约定

- 工具注册（`src/tools/**/index.ts`）使用 `translate(key)`：
  - 若 key 缺失，返回 key 本身，便于发现缺漏（需要补齐 `en.yml` 与 `zh.yml`）。

### 3.3 动态文案（校验/通知）

- 校验消息建议用函数返回：`message: () => t('...')`
- 复制提示建议传入：`notificationMessage: t('...')`

### 3.4 公共组件文案

- 复制相关组件（如 `c-text-copyable` / `InputCopyable` / `TextareaCopyable` / `SpanCopyable`）统一使用：
  - `ui.inputCopyable.copy`
  - `ui.inputCopyable.copied`
- Key-Value 列表对布尔值与空值的默认显示使用：
  - `ui.keyValueList.yes` / `ui.keyValueList.no`
  - `ui.keyValueList.notAvailable`

### 3.5 语言选择器约束

- 下拉框仅展示 `zh` 与 `en`。
- 若本地存储/历史状态存在其他 locale，会自动回落到 `zh`。
